# DECISION-GATE — the structural cure for recurring narrowness (CEO 1834/1836/1838)

## The recurring flaw (named honestly)
The main loop (the "brain") defaults to a NARROW value lens — ship-it-green +
game-feel, an engineer's single value system — and **concludes alone**: it spawns
craft agents as advisors but performs the synthesis and the final judgment itself.
CEO 1838: this narrowness is not a one-time mistake, it is a **recurring default** —
fixing one instance does nothing, because the *next new, unseen situation* gets the
same solo-narrow treatment. The cure therefore cannot be "I understood this time."
It must be **structural**: a gate that makes solo-narrow concluding impossible, that
persists across sessions, and that auto-applies to situations never seen before.

## The binding rule
**No substantive product / design / direction / priority CONCLUSION may be reached
by the main loop's solo synthesis.** Every such decision MUST route through, in order:

1. **Latest-first research** — dated 2026 sources (per the standing latest-first
   mandate). Training knowledge and the current codebase are stale-until-verified.
2. **The STANDING diverse-values council** — `.claude/workflows/diverse-values-council.js`:
   8 conflicting stakeholder value systems (paying parent, the 6yo child, 英検 teacher,
   retention strategist, art director, cultural-authenticity critic, accessibility/
   inclusion advocate, child-developmental psychologist) that each READ the accumulated
   corpus (MEMORY index + design/governance/business docs) AND today's latest web, then
   reach a **collective** conclusion with the value CONFLICTS made visible — not averaged.
3. **The relevant craft studio** — game-studio-panel / character-studio / world-studio
   for the discipline-craft layer.

**The council's collective conclusion governs.** The main loop's job is to EXECUTE it
and to SURFACE the value-conflicts to the CEO for adjudication — never to override the
council with its own taste, and never to substitute its solo synthesis for the council.

## What triggers the gate (substantive)
Cast/characters; opening/onboarding/first-run; world & narrative direction; art
direction; monetization / retention / parent-trust framing; any major UX or new-feature
DESIGN; and any "what does world-class mean here / what should we build next at the
product level" question — **including new, unseen situations the loop has not met before.**
The loop may NOT opt out by judging "this one is simple / I already know" — that
judgment IS the narrowness. When unsure whether something is substantive, it is.

## What does NOT trigger it (execution, not conclusion)
Mechanical bug fixes; test updates; applying an already-council/CEO-approved spec;
verifier/CI fixes; status reporting; reversible green-gated implementation of a
decision the council already made.

## Anti-recurrence guarantees
- **Durable:** this file + the named council workflow + a memory ([[diverse-values-council-standing]])
  persist across sessions, so the gate applies to future/unseen work automatically.
- **Standing council:** re-run the council per substantive decision; it is not a one-off.
- **Auditable:** every substantive conclusion must state which council run + which craft
  studio produced it, so solo-narrow concluding is visible if it recurs.
- **Conflicts to the CEO:** genuine value tradeoffs (e.g. errorless-fun vs scored-英検,
  defer-form vs parent-proof, named-mains vs blank-きみ) are presented for CEO
  adjudication, not resolved by the loop's fiat.
